# 主题色配置 — 功能设计文档

## 1. 模块：主题色配置

### 1.1 基础信息

| 项目 | 内容 |
| --- | --- |
| 模块名称 | 主题色配置（原「主题化」需求已收敛） |
| 端口类型 | 微信小程序（客户端）+ Web 后台管理端 |
| 目标用户 | 平台运营/管理员（配置端）；C 端车主用户（消费端，无感知配置） |
| 业务场景 | 平台在后台统一配置客户端小程序的**品牌主题色**，客户端启动或进入首页时拉取并应用，用于按钮、链接、选中态、价格强调等主色场景。 |
| 上游入口 | 后台「基本配置 > 客户端主题色」 |
| 下游影响 | 微信小程序全局/页面级 CSS 变量或主题 Token 中的 `--brand` 及相关衍生色 |
| 设计系统 | 客户端以 `功能模块设计包/02_优惠套餐/01_优惠套餐_客户端.html` 首页结构为基准；后台遵循 `DESIGN/Web后台管理端页面结构约束规范.md` |
| 关联模块 | 小程序首页、各业务 Tab 页、与品牌色绑定的组件（按钮、Tab 选中、标签、进度条等） |
| 原型文件 | `04_主题化_后台管理端.html`、`04_主题化_客户端.html`；原型 API 见 `server/` |

### 1.2 功能目标

- **平台目标**：在不改动页面布局、文案、节日皮肤的前提下，快速切换客户端**主品牌色**，满足品牌统一或短期运营配色需求。
- **用户目标**：C 端用户无额外操作；视觉仅体现为品牌色变化，交互与信息架构不变。
- **成功结果**：后台保存主题色后，客户端下次拉取配置即可生效；未拉取到配置时使用系统默认色 `#165DFF`。

### 1.3 范围与边界

#### 1.3.1 本期包含

- 后台配置项：**主题色**（HEX，如 `#165DFF`）。
- 后台提供色值输入（取色器 + 文本）、**常用色快捷选择**（仅为颜色预设，非「主题包」）、保存、重新拉取、恢复默认。
- 客户端从配置接口读取 `primaryColor`，映射为 `--brand`、`--brand-hover`、`--brand-active`、`--focus-ring` 等**主色衍生 Token**（衍生算法由前端统一实现，见开发说明）。
- 客户端在首页等关键页展示品牌色效果；原型以首页为示例页。

#### 1.3.2 本期不包含（需求变更明确排除）

- **多套主题/皮肤切换**：如春节、元旦、国庆等节日主题，或按 `themeId` 切换整站配色方案。
- **非主色 Token 批量下发**：背景色、卡片色、正文色、边框色、Hero 遮罩、轮播文案等**不随配置变更**。
- **暗色模式 / 跟随系统主题**。
- **按门店、按用户、按时段**差异化主题色。
- **客户端用户自助换肤**。
- 主题色变更的**审批流**（若需可后续单独立项）。

#### 1.3.3 边界说明

- 本模块只负责「主色配置 + 下发 + 客户端应用」；页面结构、业务文案、图片素材由各自业务模块维护。
- 与「布局调整」「金刚区」等模块独立；主题色变更不影响菜单配置数据。
- 后台管理端自身 UI 仍使用 Arco/项目后台设计 Token，**不**随客户端主题色联动。

### 1.7 页面结构与交互设计

#### 1.7.1 客户端主题色配置（后台 P01）

**页面定位**：表单型配置页，非主题列表。

**布局**（遵循 Web 后台壳层规范）：

- 面包屑：基本配置 > 客户端主题色
- 功能区：保存、重新拉取、恢复默认
- 配置区（卡片）：
  - 当前生效色：色块 + HEX 文本
  - 编辑：`<input type="color">` + HEX 文本框（双向同步）
  - 常用色：若干色块按钮（平台蓝、红、橙等），点击仅填充编辑区，**不直接生效**
  - 说明文案：仅影响客户端主按钮、链接、选中态等；不改变页面布局与节日皮肤

**交互规则**：

- 色值未变更时「保存」禁用。
- HEX 校验：`^#([0-9A-Fa-f]{6})$`；不合法 toast 提示。
- 保存成功 toast：「已保存，客户端下次拉取配置将生效」。
- 恢复默认需二次确认（Modal）。

#### 1.7.2 客户端应用（小程序）

- 在应用启动或首页 `onShow` 拉取 `GET /api/theme/color`（正式环境路径以接口文档为准；原型为 `GET /api/theme/active` 返回 `primaryColor`）。
- 将 `primaryColor` 写入根节点 CSS 变量；**不**切换 `data-theme`、**不**替换轮播文案。
- 拉取失败：使用默认 `#165DFF`，不阻断业务。

### 1.8 数据与接口建议

**配置实体（全局单例）**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| primaryColor | string | 主色 HEX，默认 `#165DFF` |
| updatedAt | datetime | 最后更新时间 |
| updatedBy | string | 操作人 |

**接口（建议）**

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/theme/color` | 返回 `{ primaryColor, defaultPrimaryColor }` |
| PUT | `/api/theme/color` | body `{ primaryColor }`，校验 HEX |

原型 `server` 为演示用途，路径暂保留 ` /api/theme/active`，响应字段为 `primaryColor`。

### 1.9 验收标准

- [ ] 后台可配置、保存、恢复默认主题色，非法色值不可保存。
- [ ] 客户端拉取成功后，按钮/Tab 选中/链接等主色与配置一致。
- [ ] 客户端接口失败时使用默认色，页面可正常使用。
- [ ] **不存在**主题 ID 列表、节日主题切换、轮播/背景随主题联动等行为。
